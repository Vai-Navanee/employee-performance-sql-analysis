# 📊 Employee Performance Analysis (SQL)

## 📌 Overview
This project analyzes employee performance, salary distribution, and job roles for ScienceQtech, a data science company. The analysis uses SQL to extract insights that support HR decision-making and workforce optimization.

---

## 🎯 Objective
To evaluate employee performance, salary structure, and experience levels using SQL queries and database techniques.

---

## 📂 Dataset
The analysis is based on multiple relational tables:
- `emp_record_table` – employee details including role, salary, and experience  
- `proj_table` – project allocation data  
- `data_science_team` – team structure and roles  

---

## 🛠 Tools & Technologies
- SQL (MySQL)
- Joins, Subqueries, Window Functions
- Stored Procedures & Functions
- Database Design & Indexing

---

## 🔍 Analysis Process
- Data extraction using SQL queries  
- Joining multiple tables for comprehensive analysis  
- Aggregation and filtering for performance insights  
- Use of advanced SQL features (window functions, procedures)  
- Query optimization using indexing  

---

## 📊 Key Insights
- Employee experience has a strong impact on salary levels  
- Higher performance ratings are associated with increased compensation  
- Salary distribution varies significantly across job roles  
- Clear role definitions improve workforce structure and analysis  
- Optimized queries improve database performance and efficiency  

---

## 💡 Business Recommendations
- Implement experience-based salary benchmarking  
- Align compensation more closely with performance metrics  
- Standardize job roles for better workforce management  
- Optimize database systems to improve reporting efficiency  

---

## 🧠 Conclusion
This project demonstrates how SQL can be used to analyze workforce data, uncover performance trends, and support data-driven HR decisions.
